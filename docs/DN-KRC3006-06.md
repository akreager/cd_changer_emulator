## 1. Purpose

This document records the known error codes used by Kenwood CD changers when communicating with compatible head units over the 13-pin mini DIN interface. These codes are displayed on the head unit (e.g., KRC-3006) but originate from the changer hardware — the head unit simply renders whatever status the changer reports over the serial data bus.

Understanding these codes is relevant to the emulator project in two ways: (1) we may need to replicate error status reporting in the emulator firmware, and (2) they informed the diagnosis of the original KDC-CX85 changer used in early project phases.

---

## 2. Changer Status

The KDC-CX85 10-disc changer originally acquired for this project produces E-04 on every disc slot when connected to the KRC-3006. Given that the error is universal across all discs (including known-good commercial pressings), the root cause is a degraded or failed laser pickup assembly (GaAlAs diode, λ=780 nm). This is consistent with the unit's age (~27 years). The CX85 has been retired from the project.

A KDC-C717 10-disc changer was acquired from a surplus auction and restored to working condition after cleaning the laser lens assembly. The C717 is now the reference changer for all protocol capture work (TP-KRC3006-01). An additional benefit of the C717 is verified CD-R and CD-RW compatibility, which avoids risking original pressed CDs during testing.

## 3. Error Code Table

The following error codes are standardized across Kenwood's CD changer product line. They apply to changers using the 13-pin mini DIN connector with both O protocol (pre-1998, round connector) and N/C protocol (post-1998, rectangular connector) head units.

| Code | Meaning | Notes |
|------|---------|-------|
| E-00 | Cable connection fault | Poor or missing connection between changer and head unit on the 13-pin umbilical. |
| E-01 | Magazine not loaded | Magazine is missing or not fully seated in the changer. |
| E-02 | Empty slot selected | The selected disc slot in the magazine contains no disc. |
| E-03 | Invalid track number | A track number was requested that does not exist on the current disc. |
| E-04 | Disc unreadable | Disc is dirty, scratched, upside-down, or the laser cannot read the TOC. When this error appears on every disc, it typically indicates a failed or degraded laser pickup rather than disc quality issues. |
| E-05 | Disc inserted upside-down | Disc is loaded label-side-down. |
| E-10 | No data on disc | MiniDisc-specific: no recorded content. Changer auto-advances to next disc. |
| E-77 | Unit malfunction | General internal fault. User action: press the reset button on the head unit. |
| E-88 | Service/diagnostic mode | Not an error. Entered via a special button sequence (hold magazine eject + press head unit reset, release after 1 second, then press CD button). Used for laser power adjustment via VR1/VR2 potentiometers. Documented in the KDC-CX85 service manual (B51-7552-00). |
| E-99 | Mechanical failure | Disc may be jammed, or the changer mechanism has a hardware fault requiring service. Sometimes referred to as "Mecha Error" in service documentation. |
| E-0d | Thermal protection | Protective circuit activated because internal temperature exceeded 60°C. Labeled "Hold Error" in some documentation. |

## 4. Error Communication Architecture

Error codes originate inside the changer and are transmitted to the head unit as status bytes over the O protocol serial bus. The head unit has no independent knowledge of changer-side conditions — it cannot detect a dirty disc, an unseated magazine, or a mechanical jam. It simply displays whatever error status the changer reports.

This means the emulator will need to define its own error reporting strategy. Since the emulator has no physical disc mechanism, most hardware-related errors (E-01, E-02, E-04, E-05, E-10, E-99) have no direct analog. The emulator may choose to report E-02 for an empty virtual disc slot, or define a mapping for software-side fault conditions.

The specific byte values used to encode these error states on the bus are not yet confirmed and will be captured during Phase 2B of TP-KRC3006-01 (changer-connected capture). The szymtro 8051-based emulator project on elektroda.pl documented some status bytes including a "no disc" state (0x3A:0x00), but a complete error byte mapping has not been published.
